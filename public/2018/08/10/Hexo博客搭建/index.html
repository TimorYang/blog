hexo

阅读本篇文章，您需要具备 Linux，Git，Nginx 等相关的知识。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

hexo安装

安装前提

  • Node.js
  • npm
  • hexo官方文档
  • 本地服务器（Mac, Windows等）
  • 远程服务器
  • 域名

安装Hexo

打开终端(Mac)或者命令行(Windows)
执行命令: npm install -g hexo-cli
等待命令执行结束，Hexo就安装完成了。

远程服务器

前面介绍了如何在本地安装和部署 Hexo ，接下来我们需要将 Hexo 生成的静态页面上传到远程服务器上，以供网友们浏览。

讲到这里我们还是先谈谈 HTTPS吧。

HTTPS

超文本传输安全协议（英语：Hypertext Transfer Protocol Secure，缩写：HTTPS，常称为HTTP over TLS，HTTP over SSL或HTTP Secure）是一种透过计算机网络进行安全通信的传输协议。

Apple、Google 一直在推进 https, 同时为了避免我们站点遇到流量劫持，建议大家建站的时候配置下 htps。

Let’s Encrypt

Let’s Encrypt由互联网安全研究小组（缩写ISRG）提供服务。主要赞助商包括电子前哨基金会、Mozilla基金会、Akamai以及思科。2015年4月9日，ISRG与Linux基金会宣布合作。
用以实现新的数字证书认证机构的协议被称为自动证书管理环境（ACME）。GitHub上有这一规范的草案，且提案的一个版本已作为一个Internet草案发布。
Let’s Encrypt宣称这一过程将十分简单、自动化并且免费。

这里我们将使用 Github 上的开源项目 acme.sh 完成 https 证书的申请，验证和自动续签。

  1. 安装
  1. 生成证书 & 验证证书

    acme.sh 实现了 acme 协议支持的所有验证协议。 一般有两种方式验证: http 和 dns 验证。
    这里笔者只介绍 http 验证，dns验证没有试过，就不发表意见了。



  1. 准备好您申请的域名

  2. 执行命令：acme.sh --issue -d mydomain.com -d www.mydomain.com --webroot /home/wwwroot/mydomain.com/
    若该命令执行中产生错误，在刚刚执行的命令后面加上 --debug
    查看错误信息，并根据错误进行修改。

  3. 注意，如果之前服务器配置过 Nginx 建议还原 nginx 配置，笔者当时就是因为这个问题被卡住了，证书一直验证不成功。提示：Verify error:Invalid response from [domain]所以要保证 80 端口不要被占用，证书验证的时候会访问当前服务器的 80 端口，如果请求成功，颁发证书的机构接收到返回，就会下发证书。

nginx

笔者在申请 https 证书的时候，被 nginx 的配置坑了不少，当时申请 vps 时，懒癌晚期，随手配置了下nginx 没有做任何记录，导致隔了5个月再来碰这些东西，完全抓虾😂。

那就具体谈谈笔者当时遇到的一些问题

  • Verify error:Invalid response from yourdomain.com

  • nginx: [emerg] bind() to 0.0.0.0:80 failed (98: Address already in use)

针对 Nginx 配置的一些建议

在 /etc/nginx/nginx.conf 中默认包含了 /etc/nginx/conf.d/*.conf 和 /etc/nginx/sites-enabled/*，而 /etc/nginx/sites-enabled/* 默认包含了 /etc/nginx/sites-available/, 所以我们在配置站点时，最好在 /etc/nginx/sites-available/
建立新的站点配置文件，然后 link 到 /etc/nginx/sites-enabled/*。

link 命令

sudo ln -s /etc/nginx/sites-available/xxx /etc/nginx/sites-enabled/xxx
